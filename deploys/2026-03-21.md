- git post-receive hook：`git push prod main` 自动触发部署

## 遇到的问题和解决

| 问题 | 原因 | 解决 |
|------|------|------|
| `git pull` 超时 | 阿里云服务器无法访问 GitHub git 协议 | 改为 push-to-deploy，本地 push 到服务器 |
| PM2 启动报 `Cannot find module 'dotenv/config'` | releases 目录只有 dist，没有 node_modules | 在 releases 中创建 node_modules 软链指向源码目录 |
| git hook 覆盖新版 deploy.sh | hook 执行 `git checkout -f` 把服务器上的新脚本还原成 git 里的旧版本 | 将新版 deploy.sh 提交到 git 仓库 |
| 登录/注册报 `Access denied (using password: NO)` | releases 目录没有 `.env`，PM2 以 `current/backend` 为 cwd 启动时读不到数据库密码 | 在 releases 中创建 `.env` 软链指向 `~/app/backend/.env` |

## ⚠️ 下次部署前必须检查

每次新建 release 目录时，以下三个软链缺一不可：

```bash
releases/$VERSION/backend/node_modules  → ~/app/backend/node_modules
releases/$VERSION/backend/.env          → ~/app/backend/.env
releases/$VERSION/frontend/node_modules → ~/app/frontend/node_modules
```

deploy.sh 已包含这三条软链创建逻辑（step 5），**不要手动跳过或修改这一步**。
