## 使用流程

### 1. 开始开发新模块前

```bash
# 1. 查看总体需求
cat 00-总体需求.md

# 2. 创建子模块需求文档（如果不存在）
# 使用上面的模板创建，例如：
# vim 01-用户认证系统.md

# 3. 详细描述该模块的需求、API、数据库设计等
```

### 2. 开发过程中

- 参考需求文档进行开发
- 遇到需求不明确的地方，及时更新文档
- 发现需求变更，及时更新文档

### 3. 开发完成后

- 检查是否满足所有验收标准
- 更新文档中的完成状态
- 记录遇到的问题和解决方案

## 文档维护

### 更新原则
- 需求变更时必须更新文档
- 发现文档错误及时修正
- 添加实际开发中的经验和注意事项

### 版本控制
- 所有文档纳入Git版本控制
- 重大变更在文档中记录变更历史

## 注意事项

1. **需求文档是开发的依据**，开发前必须先有需求文档
2. **需求文档要详细具体**，包含API设计、数据库设计、验收标准
3. **需求文档要及时更新**，保持与实际开发同步
4. **需求文档要易于理解**，使用清晰的语言和示例
