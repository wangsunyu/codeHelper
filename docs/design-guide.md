# UI设计指南 - Pencil设计方案

## 概述

本项目采用**Pencil设计工具**进行UI设计，实现需求→设计→代码的全自动化工作流。

## 为什么选择Pencil

1. **完全自动化**：Claude可以直接操作Pencil设计文件，无需人工介入
2. **设计即代码**：设计和代码100%一致，零还原度问题
3. **快速迭代**：需求变更可在10-20分钟内完成设计和代码的同步修改
4. **完整闭环**：需求 → 设计 → 代码 → 迭代，全程自动化

## Pencil设计工作流

```
第1步：生成项目需求
├─ Claude生成详细需求文档
├─ 功能列表和用户故事
└─ 技术选型和架构设计

第2步：生成设计页面和风格
├─ Claude使用Pencil生成初版设计
├─ 应用风格指南（现代、简洁、科技感）
├─ 生成所有核心页面
└─ 导出设计截图供确认

第3步：确认风格并生成详细需求
├─ 用户确认设计风格和配色
├─ Claude生成各功能的详细需求
└─ 完善设计细节

第4步：根据详细需求修改设计
├─ Claude根据需求调整设计
├─ 添加/修改组件和布局
└─ 保持设计系统一致性

第5步：需求迭代自动同步
├─ 需求变更 → Claude自动修改Pencil设计
├─ Claude自动生成/修改React代码
└─ 设计和代码始终保持同步
```

## 用户需要提供什么

当你说"帮我设计"时，请提供以下信息：

### 1. 设计范围（必需）
```
- 需要设计哪些页面？
  例：首页、MCP列表页、详情页、用户中心

- 或者是修改现有页面？
  例：在详情页添加评分功能
```

### 2. 风格偏好（可选，有默认值）
```
- 整体风格：现代/简约/科技感/温暖/专业（默认：现代简约）
- 主色调：蓝色/绿色/橙色/紫色（默认：蓝色）
- 目标用户：开发者/普通用户/企业用户（默认：开发者）
```

### 3. 特殊要求（可选）
```
- 参考网站：例如"类似GitHub的风格"
- 特殊功能：例如"需要支持暗色模式"
- 布局偏好：例如"侧边栏导航"或"顶部导航"
```

### 4. 设计优先级（可选）
```
- 优先设计哪些页面？
- 哪些功能是核心功能？
```

## 最简单的使用方式

如果你不确定提供什么信息，只需说：

```
"帮我设计首页"
"帮我设计MCP列表页"
"帮我设计整个平台的UI"
```

Claude会：
1. 使用默认的设计风格（现代简约 + 蓝色主题）
2. 基于项目需求文档生成设计
3. 生成设计截图给你确认
4. 根据你的反馈调整

## 设计输出物

Claude使用Pencil设计后会提供：

1. **设计文件**：`.pen`格式的设计源文件
2. **设计截图**：PNG格式的页面预览图
3. **设计说明**：设计思路和关键决策说明
4. **React代码**：基于设计生成的前端代码（Tailwind CSS）

## 设计迭代

需求变更时，只需告诉Claude：

```
"把主色调改成橙色"
"调整首页的布局，把特色卡片改成4列"
```

Claude会：
1. 自动修改Pencil设计文件
2. 生成新的设计截图
3. 同步修改React代码

## 设计版本管理

### 版本号规则

采用语义化版本号：`v{major}.{minor}.{patch}`

- **Major（主版本）**：重大设计改版，风格或布局大幅变化
  - 例：v1.0.0 → v2.0.0（从蓝色主题改为橙色主题）

- **Minor（次版本）**：新增页面或重要功能
  - 例：v1.0.0 → v1.1.0（新增评论功能）

- **Patch（补丁版本）**：小修改、优化、bug修复
  - 例：v1.1.0 → v1.1.1（调整按钮间距）

### 版本记录内容

每个版本需要记录：

1. **版本号**：v0.1.0
2. **发布日期**：2026-03-18
3. **变更类型**：新增/修改/优化/修复
4. **变更内容**：具体改动说明
5. **影响范围**：哪些页面/组件受影响
6. **截图对比**：变更前后的对比图（重要变更）
7. **代码同步**：是否已同步到前端代码

### 版本记录示例

```markdown

### 新增
- 首页设计（homepage.pen）
```

### 版本归档流程

当发布新版本时：

1. **创建版本记录**
   ```bash
   # 在 designs/versions/DESIGN_VERSIONS.md 中添加版本记录
   ```

2. **归档当前版本**
   ```bash
   # 将 designs/current/ 复制到 designs/archive/v0.1.0/
   # 将 designs/screenshots/ 复制到 designs/archive/v0.1.0/screenshots/
   ```

3. **更新当前版本**
   ```bash
   # 在 designs/current/ 中进行新版本的设计修改
   # 更新 designs/screenshots/ 中的截图
   ```

4. **同步代码**
   ```bash
   # 根据新设计更新 frontend/ 中的代码
   # 确保设计和代码一致
   ```

### 版本回滚

如果需要回滚到历史版本：

1. 从 `designs/archive/v0.x.x/` 恢复设计文件
2. 从对应的Git提交恢复前端代码
3. 在版本记录中标注回滚操作

## 注意事项

1. **设计文件是加密的**：`.pen`文件只能通过Pencil MCP工具访问，不要使用Read或Grep工具读取

2. **设计和代码同步**：修改设计后，Claude会自动同步修改代码

3. **保持一致性**：所有页面使用统一的设计系统和组件库

4. **响应式设计**：所有设计自动支持移动端、平板和桌面端

5. **版本管理**：每次重要变更都要创建新版本并记录

6. **截图更新**：设计修改后要及时更新截图

7. **文档同步**：版本记录要与实际设计文件保持同步
