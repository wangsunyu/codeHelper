# 生产环境版本更新部署（含备份与一键回滚）— Proposal

## 背景

V1 core MVP 已完成 bug 修复，需要安全地将新版本推送到生产服务器（39.103.65.215）。
当前部署流程没有系统性的备份机制和回滚方案，一旦新版本出现问题，恢复困难。

## 要做什么

1. **部署前备份**：自动备份当前运行的前端构建产物、后端代码、MySQL 数据库
2. **版本化部署**：将每次部署打上时间戳标签，保存到服务器 `releases/` 目录
3. **一键回滚**：提供 `rollback.sh` 脚本，可回滚到任意历史版本（代码 + 数据库）
4. **更新现有 `deploy.sh`**：在原部署流程中嵌入备份和版本标记逻辑
5. **本地快捷指令**：通过 `npm run deploy` 或 `make deploy` 从本地触发完整部署

## 为什么这样做

- **安全第一**：Bug 修复期间频繁部署，有可能引入新问题，必须有退路
- **数据库备份**：数据比代码更难恢复，每次部署前强制备份 MySQL
- **最小侵入性**：在现有 `deploy.sh` 基础上扩展，不引入新的 CI/CD 平台
- **可审计**：每个版本保留部署记录，方便排查问题

## 不做什么

- 不引入 Docker / Kubernetes / GitHub Actions（保持轻量）
- 不实现多环境（staging）管理
- 不做数据库迁移的自动化回滚（需手动处理 schema 变更）
- 不做前端 CDN 分发
